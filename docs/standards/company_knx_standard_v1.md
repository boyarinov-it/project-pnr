# Company KNX Standard v1

## Назначение

Этот документ фиксирует текущий стандарт компании по:
- главным группам KNX
- правилам именования
- правилам адресации
- правилам ETS CSV экспорта
- обязательным полям для генерации
- профилям кодировок экспорта

Документ является baseline для backend/domain logic.
UI не должен хранить эти правила.

---

## Общие принципы

1. Один проект = один инженерный источник правды.
2. Бизнес-правила генерации живут в backend/domain.
3. Excel/VBA используется как baseline текущей логики, но не как целевая архитектура.
4. ETS export — последний шаг после validation.
5. Все главные группы KNX являются частью стандарта компании и не задаются произвольно.

## Освещение (1/x/y)

### Reserved addresses

Адреса `1/x/0` ... `1/x/4` зарезервированы.

### Старт типовых адресов

Для групп света типовые адреса начинаются с `1/x/5`.

### Количество адресов на группу света

На одну группу света выделяется 5 адресов в порядке:

1. Вкл/Выкл
2. Диммирование относительное
3. Яркость %
4. Статус
5. Статус %

### DPT для света

| Функция | DPT |
|---|---|
| Вкл/Выкл | DPST-1-1 |
| Димм | DPST-3-7 |
| Яркость% | DPST-5-1 |
| Статус | DPST-1-1 |
| Статус% | DPST-5-1 |

### Именование света

Формат имени для ETS CSV света:

`_<group_code>__<function_label>_<floor.room.subroom>-<group_name>`

Пример:

`_1__Вкл_1.1.Прихожая-Бра`

### Description света

Формат description:

`<device_type> <device_address> Выход:<device_output><optional_dimmer> Нагрузка:<load_type>`

Где:
- `<optional_dimmer>` = ` - Диммер: <dimmer_channel>`, если канал диммера указан

Примеры:
- `RELAY 1.1.1 Выход:A1 Нагрузка:RELAY`
- `DALI 1.1.12 Выход:1 - Диммер: 2 Нагрузка:DIM`

### Иерархия ETS CSV для света

CSV должен содержать:
1. служебную строку главной группы
2. служебную строку средней группы
3. строку `Выключение всего освещения`
4. строки `Мастер резерв`
5. далее строки групп света

### Группировка по средней группе

- 50 групп света на одну среднюю группу
- 5 адресов на одну группу света
- старт третьего уровня: 5

## ETS CSV export

### Базовые правила

1. Экспорт выполняется только после validation.
2. В export не должны попадать неполные группы.
3. Export history обязателен.
4. Export file должен быть связан с ProjectRevision / ExportJob.
5. CSV writer не должен содержать бизнес-логику адресации.

### Кодировки

Поддерживаемые кодировки:
- `cp1251`
- `utf-8-sig`

Для ETS lighting export по baseline по умолчанию использовать:
- `cp1251`

### Разделитель

Для ETS CSV baseline света используется:
- `,`

### Заголовок колонок

Для ETS lighting baseline по VBA:
- обычная header-строка с названиями колонок не используется

---

## Обязательные поля для lighting export

Для группы света обязательны:
- `project`
- `room`
- `room.code`
- `group.code` (числовой)
- `group.name`
- `load_type`
- `device_type`
- `device_address`
- `device_output`

Опционально:
- `dimmer_channel`

Если обязательные поля не заполнены:
- export блокируется validation error
